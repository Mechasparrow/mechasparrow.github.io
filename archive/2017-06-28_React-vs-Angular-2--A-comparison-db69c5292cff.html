React vs Angular 2: A comparison

If you are part of the Web programming community, you might have heard of React and Angular 2. They are both Front-end web app frameworks.

They are similar and different in a couple of ways.

In this post, I will cover similarity and differences between both frameworks.

Similarities

I’ll start kind of in reverse with similarities first. Two big features that they both share are Native Application Development and Components.

Components are objects that contain certain functionality. These objects can later interact with similar objects to form a large application.

Native Application Development is the development of applications for a specific device. Some examples of this include Desktop Applications and Android/IOS applications.

React offers this through its React Native library. You can use the knowledge gained from learning React to create Android and IOS applications.

Angular 2 offers this through the

  • Ionic Framework
  • NativeScript

Both web frameworks can be used to create desktop applications through the Electron library.

React Specific

React is officially defined as a “declarative, efficient, and flexible JavaScript library for building user interfaces (https://facebook.github.io/react).” This means that it is good at the visual aspect of web applications.

You can think of it as the V in MVC. Everything else is left to the developer.

React is good at rendering HTML through what they call the VirtualDOM with the usage of JSX. JSX is an XML like templating language that React uses to render your UI.

You can also update the UI by updating its, “state”. The state in React is the data that you are telling React to render.

Angular Specific

Angular is defined as a “platform that makes it easy to build applications with the web. (https://angular.io/docs)” Angular has many features and libraries to help do this for developers.

With all the libraries combined, you’re looking at a *VC framework. The Model layer is still kind of optional. It’s up to you to incorporate it.

Angular is a modular framework which contains components. These components contain JavaScript logic interacts with an HTML template.

Each Angular component can have HTML template attached. This allows developers to create Single Page Applications.

Angular also has the capacity to be used to create Progressive Web Apps. This can be done through its supplementary frameworks or manual configuration.

Conclusion

React is good if you want to code and you are only worried about getting the end result to the screen.

Angular is good if you want a framework that handles more of the grunt work. It also has an software architecture designed to handle larger scale web applications.

So… It really depends.
