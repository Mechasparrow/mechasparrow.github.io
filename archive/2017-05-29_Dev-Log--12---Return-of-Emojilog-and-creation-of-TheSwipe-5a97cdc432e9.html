Dev Log #12 — Return of Emojilog and creation of TheSwipe

Hey, everybody. It is another dev log. This is the first summer edition too!

After high school finals, I’ve been able to put more time into working on my side projects. I’ve been able to work on TheSwipe, Emojilog, and redesigning my portfolio website.

All in all a great development weekend. Let’s go into each with a little more detail.

We will also discuss future projects as well.

Website Redesign!

This is the first thing I tackled the day after high school finals ended.

I was looking at several other Jekyll websites. I realized that my website could look cleaner if I used a Jekyll template as a foundation.

So I found a Jekyll template with nice aesthetics. The one I used has the name, Minimal Mistakes. This led to a clean look with a more organized portfolio at http://michael.navazhylau.com/work

The main migration that took the most time was moving the projects. This is since the list was stored as a JSON file. I had to create a Markdown file for each project on the portfolio. It paid off.

The website looks nice and is quite simple to navigate.

View It at: http://michael.navazhylau.com

TheSwipe

The main goal of TheSwipe is to allow writers to research and pull information discovered.

I haven’t used Ruby on Rails for a while, so I decided to use it for this project. I could have built the entire application on the front-end.

Even still, I was able to get back into the swing of things back-end wise as a result of this side-project. Currently, the main ability it has is adding links, text, and tags.

The next feature will be tag search. This should take less than an hour to complete.

I’ll link to the project once I get it up on Heroku. For now, you can view the source code at https://github.com/Mechasparrow/theswipe

Emojilog

Emojilog is a journal application for logging your emotions with emoji on the daily.

The main thing that I changed to the application a couple days back is the storage code.

It used to use localforage. I realized through reading the ionic docs that I could use IonicStorage instead. This is exactly what I did.

I also did some experimentation with push notifications. I found a service that simplifies the push notification process. It is called OneSignal. I can’t wait to integrate it into Emojilog.

The next feature to be put in place is sentiment analysis. This is when emotions and opinions behind a block of text are identified by a computer.

By implementing this into Emojilog or by separating it into a different project, I can figure out how a person’s day went. This could also lead to staggering conclusions to a person’s attitude and personality.

It also should be taken into consideration that computers are not 100% accurate. This means that false results could occur.
