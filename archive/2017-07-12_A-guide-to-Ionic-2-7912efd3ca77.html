A guide to Ionic 2

Want to get started building mobile applications but don’t know where to start? Well, if you’ve done web development previously then Ionic is a good option.

Ionic is a framework built for creating mobile apps with web technologies such as Angular, TypeScript, and HTML.

Here’s an introduction to the framework.

Introduction

Ionic 2 is a framework for building mobile applications with JavaScript. It is built on top of the Angular and Cordova frameworks.

Its architecture is built with components. These components save developer and design time.

It can cross-deploy to Android, IOS, and the browser. This makes the development experience nicer instead of having to port the same app 3 separate times.

Getting Started

To start working with Ionic make sure that you have nodejs and npm installed. After that, we will install ionic and cordova.

npm install -g ionic cordova

after that we create our first app

ionic start ourApp — blank

This will create a blank ionic 2 app. We can now run the app with

cd ourApp
ionic serve

Folder structure

There will be quite a bit of code once the application is generated. The main app source code can be viewed inside ./src/app/ and ./src/pages

The app folder is where you make declarations for your app.

The pages folder is where the views for your app are. Each page contains an HTML, CSS, and TS file.

The HTML file makes use of the Ionic’s visual components. Ionic has a large assortment of visual components to choose from.

This makes the app look nice on several devices.

./src/assets are used for media assets relating to your app.

../src/theme is used when you want to change the default theme that ionic provides for your app.

Coding

After becoming familiar with folder structure, it is largely just coding.

I would highly recommend learning Angular and TypeScript as they are both used when developing an app with Ionic.

After learning Angular, the rest of Ionic should not be too hard to understand.
