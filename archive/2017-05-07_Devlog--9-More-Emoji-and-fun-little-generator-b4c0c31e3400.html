Devlog #9 More Emoji and fun little generator

Hey, everyone. It’s time for another development log. After seeing the initial reaction of Emojilog, I’ve been adding more features.

I have not stopped working on smaller side projects though. I’ll talk about those as well.

How’s Emojilog?

Well, It has gained some wonderful viewership on Hacker News and Reddit /r/SideProject. The only issue is that I neglected to think of how I would keep users.

The first thing that I did after I felt that the idea was validated was porting it over.

I used VueJS to build the app. In order for me to build it as a PWA (Progressive Web App), I would have to use something like Ionic to get the job done.

I was able to get the port done in a day after school.

Several days after the initial launch, I created a mailing list popup and link. I’ve been gaining more emails by the day. That’s the marketing aspect of it.

You can check out Emojilog at https://emojilog.surge.sh.

Another toy app!

GitHub does this cool thing where it auto-generates a project name for you. I decided that I would like to figure out how easy it would be to create one. Luckily, I already had most of the code I need from an earlier application I created. I switched out the data files and the template code and I was set. You can see the project name generator at https://github.com/Mechasparrow/project-name-gen
