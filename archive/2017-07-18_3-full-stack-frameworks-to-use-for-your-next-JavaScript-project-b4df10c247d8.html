3 full stack frameworks to use for your next JavaScript project

Hey, everyone. It’s Mechasparrow again (a.k.a Michael Navazhylau). I’ve started working a bit with full stack JavaScript.

As a result, I’ve been doing some research on full stack JavaScript frameworks.

I’d like to share 3 of these frameworks I’ve seen with some pros and cons of each.

These frameworks are MEAN, MeteorJS, and Sails.js

MEAN

The MEAN stack is one of the most common JavaScript full stack frameworks out there. It is built on top of MongoDB, Express, AngularJS, and Node.js.

MongoDB is a NoSQL database system. Saves data in JSON (hmm?)

Express is a very light web framework for building web applications in Node.

AngularJS is a front-end JavaScript framework developed by Google. It offers cool features such as two-way data binding, templating, and more. Allows for fast paced front end development

Node.js allows for running server side JavaScript.

So what are the pros and cons to this full stack setup?

Pros:

  • Express is are very flexible web framework. This leaves most of the tooling and library selection to the developer.
  • Mongo is usually easier to work with and works well with javascript natively
  • Angular provides powerful abstraction for the creation of single page applications and dynamic websites

Cons:

  • MongoDB can get complex as it is not really intended to work with bigger sets of data
  • Requires higher level knowledge of JavaScript

Meteor

Meteor is a full stack, opinionated MVC framework for Node.js. It’s a little less flexible than a lightweight framework like ExpressJS. It can really help accelerate development with IOS, Android, and web applications.

Pros:

  • Swappable front-end (you can use React or Angular instead of Blaze)
  • Pre-assembled compatible part (default tech for each layer of the stack)
    Fast development cycles (scaffolding, packages, libraries)

Cons:

  • Only integrates with a MongoDB database (no support for SQL databases)
  • If the application becomes large you may need a different stack than Meteor

Sails.js

Sails.js is an MVC web framework written in JavaScript. It includes a command line generator. It also has many similarities with the Ruby on Rails framework. It is also built on top of express and socket.io

Pros:

  • MVC pattern (commonly understood by most web developers)
  • Includes essentials such as WebSockets, pipeline management, database ORM

Cons:

  • Weak documentation
  • Has very little room for customization

Conclusion

The 2 best options out of these 3 frameworks seem to be Meteor and the MEAN stack. It is really beneficial to know JavaScript well to best utilize these frameworks.
